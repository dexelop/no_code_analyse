# 📊 분개장 기반 AI 재무 예측 및 보정 시스템 기획서

## 1. 프로젝트 개요
* **목표:** 전산 ERP화 된 분개장 데이터를 분석하여, 단순 결산이 아닌 **'미래 손익 예측(Landing)'** 및 **'경영 컨설팅'** 리포트 제공.
* **핵심 가치:** 데이터의 공백(결측치)을 AI가 능동적으로 발견하고, 보정 제안을 통해 인간의 의사결정을 돕는 **Human-in-the-loop** 프로세스 구축.

## 2. 데이터 정의 및 역할
### 2.1. 기준 데이터 (Reference)
* **소스:** 전년도(2024년) 완료된 분개장 및 합계잔액시산표.
* **역할:** * 월별 고정비/변동비 패턴 학습.
    * 계절성(Seasonality) 이벤트(예: 12월 접대비 급증) 식별.
    * **[주의]** 12월 31일 자 '결산 분개(손익 대체, 집합 손익)'는 노이즈이므로 반드시 제거 후 학습할 것.

### 2.2. 분석 데이터 (Target)
* **소스:** 금년도(2025년) 진행 중인 분개장.
* **특징:** * 입력 시차(Time-Lag) 존재 (발생일과 입력일의 괴리).
    * 특정 기간(예: 7월~11월) 데이터가 통째로 누락되거나 입력이 지연된 상태.

## 3. 핵심 분석 및 예측 로직 (Algorithm)
### 3.1. 결측치(Gap) 보정 전략
* **고정비 (Fixed Cost):** * 임차료, 통신비, 세무기장료 등.
    * **Rule:** 전월 데이터가 금월에 없으면 **'누락'**으로 간주하고 전월 금액 그대로 자동 생성 제안.
* **변동비 (Variable Cost):** * 소모품비, 포장비, 운반비 등.
    * **Rule:** 전년도 **'매출액 대비 비용 비율(%)'**을 적용하여, 금년 매출액에 비례한 예상 비용 산출.
* **계절성 비용 (Seasonality):**
    * 접대비, 복리후생비(회식), 수도광열비(냉난방).
    * **Rule:** 단순 평균이 아닌, **'전년 동월 실제 발생액'** 또는 **'가중치'**를 적용하여 보수적으로 예측.

### 3.2. 분석의 깊이 (Granularity)
* **거래처별 분석 지양:** 편의점 A vs B 구분은 무의미함.
* **계정 과목(Account Title) 중심:** 비용의 성격(식대, 소모품비 등)에 집중하여 거시적 추세 파악.

### 3.3. 재무제표 구분 (Structure)
* **손익계산서(IS):** 수익성 분석 (매출 - 비용 = 이익). 
* **재무상태표(BS):** 현금흐름 및 건전성 분석 (자산, 부채).
* **Cross Check:** 손익계산서상 이익이 나더라도, 재무상태표상 현금/미지급금 흐름을 통해 **'흑자 도산'** 가능성 경고.

## 4. 컨설팅 및 리포팅 프로세스 (User Scenario)
1.  **진단 (Diagnosis):** * "현재 2025년 판관비율이 29%로 비정상적으로 낮습니다. 약 6,000만 원의 비용 누락이 의심됩니다."
2.  **제안 (Proposal):** * "누락된 기간에 대해 전년도 패턴을 적용해 보정할까요?" (보정 테이블 제시)
3.  **컨펌 (Confirmation - Human Touch):** * 사용자가 보정치 확인 및 수정 (예: "12월 접대비는 200만 원 그대로 반영해.")
4.  **예측 (Forecasting):** * "보정 후 예상되는 연말 당기순이익은 OOO원이며, 예상 세금은 OOO원입니다."

## 5. 주의사항 (Constraints)
* **입력 지연의 착시:** 현재 조회된 이익은 '비용이 입력되지 않아서' 생긴 가짜 이익일 수 있음을 항상 경고할 것.
* **결산 분개 필터링:** 원장 데이터 로드 시, `결산`, `대체`, `손익` 키워드가 포함된 12/31일자 시스템 자동 전표는 분석에서 제외해야 함.
* **보수적 접근:** 자금 계획 목적의 예측은 항상 비용을 넉넉하게(Max 값) 잡는 시나리오를 우선할 것.
